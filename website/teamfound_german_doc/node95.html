Return Codes

Die Return-Codes stehen immer in den Tags return-valuexx/return-value. Die Description ist optional.

0
Alles ok

1
Fehler, konnte URL nicht finden

2
Ungültige Anfrage (Pflicht-Parameter fehlen oder haben die Länge null)

3
Inkompatible Interface-Version (die Anfrage hat eine Interface-Version benutzt die der Server nicht unterstützt)

4
Kategorie existiert schon (beim hinzufuegen einer neuen Kategorie)

5
Kategorie nicht gefunden (beim suchen nach einer bestimmten Kategorie)

-1
Anderer Fehler

Die Return-Descriptions sind frei wählbar, sollten aber dem Return-Code semantisch entsprechen ;-)
